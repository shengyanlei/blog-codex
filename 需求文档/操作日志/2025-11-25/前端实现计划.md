# 前端实现计划 - 后端功能对接

根据后端已完成的 RBAC、分类管理和版本控制功能，前端需要进行相应的页面开发和功能增强。

## 1. 管理后台 (Admin Dashboard)

### 1.1 角色与权限管理 (RBAC)
**目标**: 实现基于角色的访问控制管理界面。

*   **角色列表页 (`/admin/roles`)**
    *   展示角色列表（名称、代码、描述、创建时间）。
    *   操作：新增、编辑、删除、分配权限。
*   **角色编辑/创建模态框**
    *   基本信息：名称、代码、描述。
    *   **权限分配树**：展示所有权限（按分组 `post`, `category`, `user`, `system` 分类），支持勾选分配。
*   **用户管理增强 (`/admin/users`)**
    *   在用户列表或详情页增加**角色分配**功能。
    *   支持为用户指定一个或多个角色。

### 1.2 分类管理 (Category Management)
**目标**: 实现多层级分类的管理。

*   **分类列表页 (`/admin/categories`)**
    *   **树形表格展示**：直观展示分类的层级结构（父子关系）。
    *   展示字段：名称、Slug、描述、排序、文章数（可选）。
    *   操作：新增子分类、编辑、删除、移动（修改父级）。
*   **分类编辑/创建模态框**
    *   字段：名称、Slug、描述、排序。
    *   **父分类选择器**：下拉树形选择器，用于指定父分类（不能选择自己或子节点作为父节点）。

### 1.3 文章管理增强 (Post Management)
**目标**: 对接后端的分类、元数据和版本控制。

*   **文章编辑器 (`/admin/posts/edit/:id`)**
    *   **分类选择**：支持选择文章所属分类（多选或单选，视业务逻辑而定，后端支持多对多）。
    *   **元数据编辑**：提供界面编辑 JSON 元数据（如 SEO 关键词、自定义属性）。
    *   **发布设置**：关联后端 `Post` 实体的状态流转。
*   **版本历史 (`/admin/posts/:id/versions`)**
    *   展示文章的版本列表（版本号、修改人、修改时间、备注）。
    *   **版本对比**：对比当前版本与历史版本的差异（展示 `contentDiff` 或重新计算）。
    *   **回滚操作**：将文章恢复到指定版本（调用后端接口）。

## 2. 前台博客展示 (Blog Frontend)

### 2.1 文章详情页
*   **信息展示**：显示文章的分类、标签、作者、发布时间。
*   **统计数据**：展示浏览量 (`viewCount`)、点赞数 (`likeCount`)、评论数 (`commentCount`)。
*   **交互**：点赞按钮（对接点赞接口）。

### 2.2 分类归档页 (`/category/:slug`)
*   根据分类 Slug 展示该分类下的文章列表。
*   支持层级导航（面包屑导航：首页 > 父分类 > 子分类）。

## 3. 技术栈与组件建议

*   **状态管理**: 使用 Pinia 管理用户信息、权限列表和全局配置。
*   **UI 组件库**: 继续使用现有组件库（如 Ant Design Vue 或 Element Plus，视项目当前选型而定），利用其 Tree 和 Table 组件实现分类树。
*   **权限控制**: 实现指令级权限控制（如 `v-permission="'post:delete'"`），控制按钮的显示/隐藏。

## 4. 开发优先级

1.  **P0**: 分类管理（文章发布的基础）、文章编辑器增强（分类选择）。
2.  **P0**: 角色与权限管理（系统安全基础）。
3.  **P1**: 前台展示增强（分类、统计）。
4.  **P2**: 文章版本历史与回滚。
